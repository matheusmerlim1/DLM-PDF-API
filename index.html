Como o DLM-PDF funciona

O DLM-PDF é um sistema de gestão de posse digital de e-books usando blockchain. Cada cópia de livro é um ativo único na blockchain. O leitor precisa provar que é o dono da licença antes de receber a chave para decriptografar o PDF.

Arquitetura dos 3 projetos

O sistema é composto por três projetos independentes:

┌─────────────────────────────────────────────────────────────────┐
│  DLM PDF API  (este site)                                        │
│  Back-end + documentação. Toda a criptografia acontece aqui.     │
│  URL: https://dlm-pdf-server-production.up.railway.app/api/v1   │
└────────────────────────┬────────────────────────────────────────┘
                         │  APIDLM.*  (js/api.js)
           ┌─────────────┴────────────┐
           ▼                          ▼
┌──────────────────┐      ┌───────────────────────┐
│  Livraria DLM    │      │  DLM-PDF Encriptador   │
│  Editora gera    │      │  Leitor abre .dlm      │
│  arquivos .dlm   │      │  (decriptografia via   │
│  (publisher)     │      │   MetaMask + servidor) │
└──────────────────┘      └───────────────────────┘

Regra de ouro: toda lógica de criptografia fica nesta API. Os outros dois projetos apenas consomem APIDLM.* via js/api.js.

Fluxo de autenticação e leitura

1
Conectar carteira (MetaMask)

O browser pede permissão ao MetaMask para acessar o endereço da carteira do usuário.

2
Solicitar desafio

GET /auth/challenge?address=0x... — servidor gera um nonce único com validade de 5 minutos.

3
Assinar com a carteira

MetaMask mostra a mensagem ao usuário. A assinatura prova a posse da chave privada — sem senha, sem cadastro.

4
Login — receber JWT

POST /auth/login — servidor verifica a assinatura e emite um token JWT (validade 1h).

5
Decriptografar o .dlm

POST /decrypt — envia o arquivo .dlm + assinatura MetaMask, servidor valida posse e retorna o PDF em Base64.

6
Renderizar em memória

O cliente usa PDF.js para renderizar o PDF em canvas. O conteúdo nunca vai ao disco.

Formato do arquivo .dlm

O .dlm é um PDF criptografado com um cabeçalho proprietário. Suporta versões v1, v2 e v3:

┌──────────────────────────────────────────────────────────────┐
│ v1 (60 bytes de cabeçalho)                                    │
│ MAGIC      4 bytes  "DLM\x01"                                 │
│ licenseId  8 bytes  uint64 big-endian                         │
│ padding   16 bytes                                            │
│ IV        16 bytes  vetor de inicialização AES                │
│ HMAC      32 bytes  integridade SHA-256                       │
│ ciphertext N bytes  AES-256-CBC (conteúdo do PDF)             │
├──────────────────────────────────────────────────────────────┤
│ v2/v3 (102 bytes de cabeçalho)                                │
│ MAGIC      4 bytes  "DLM\x02" ou "DLM\x03"                   │
│ licenseId  8 bytes  uint64 big-endian                         │
│ ownerAddr 20 bytes  endereço Ethereum do proprietário         │
│ verifyCode 6 bytes  (v3) código de verificação                │
│ reserved  16 bytes                                            │
│ IV        16 bytes  vetor de inicialização AES                │
│ HMAC      32 bytes  integridade SHA-256                       │
│ ciphertext N bytes  AES-256-CBC (conteúdo do PDF)             │
│ [metadata]          plaintext "DLMm{json}" ao final (v3)      │
└──────────────────────────────────────────────────────────────┘

A chave AES nunca está no arquivo. Ela é derivada por HKDF-SHA256 da chave mestra do servidor + licenseId. Sem a licença válida, o arquivo não abre.

Passo 2 — Autenticar com MetaMask

const API_BASE = 'https://dlm-pdf-server-production.up.railway.app/api/v1';

async function loginComMetaMask() {
  // 1. Solicita acesso à carteira
  const accounts = await window.ethereum.request({ method: 'eth_requestAccounts' });
  const address  = accounts[0].toLowerCase();

  // 2. Pega o desafio do servidor
  const res1    = await fetch(`${API_BASE}/auth/challenge?address=${address}`);
  const { message } = await res1.json();

  // 3. Assina (abre popup MetaMask)
  const signature = await window.ethereum.request({
    method: 'personal_sign',
    params: [message, address],
  });

  // 4. Faz login e recebe o JWT
  const res2  = await fetch(`${API_BASE}/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ address, message, signature }),
  });
  const { token } = await res2.json();

  return { token, address };
}

curl https://dlm-pdf-server-production.up.railway.app/api/v1/auth/challenge?address=0xSEU_ENDERECO

# 2. (assinar a mensagem com MetaMask ou ethers.js)

# 3. Login
curl -X POST https://dlm-pdf-server-production.up.railway.app/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"address":"0x...","message":"DLM:decrypt:...","signature":"0x..."}'

Passo 3 — Decriptografar e renderizar um .dlm

// Usando APIDLM via window.DLM (js/api.js carregado)
const { APIDLM } = window.DLM;

async function abrirDLM(dlmFile, walletAddress) {
  // Converte arquivo para base64
  const buffer  = await dlmFile.arrayBuffer();
  const bytes   = new Uint8Array(buffer);
  let   binary  = '';
  for (const b of bytes) binary += String.fromCharCode(b);
  const dlmBase64 = btoa(binary);

  // Assina com MetaMask
  const licenseId = /* leia do cabeçalho do .dlm */;
  const message   = `DLM:decrypt:${licenseId}:${Date.now()}`;
  const signature = await window.ethereum.request({
    method: 'personal_sign', params: [message, walletAddress]
  });

  // Decriptografa no servidor
  const result = await APIDLM.decrypt(dlmBase64, walletAddress, signature, message);

  // Renderiza com PDF.js
  const pdfBytes = Uint8Array.from(atob(result.pdfBase64), c => c.charCodeAt(0));
  const pdf      = await pdfjsLib.getDocument({ data: pdfBytes.buffer }).promise;
  for (let n = 1; n <= pdf.numPages; n++) {
    const page     = await pdf.getPage(n);
    const viewport = page.getViewport({ scale: 1.4 });
    const canvas   = document.createElement('canvas');
    canvas.width = viewport.width; canvas.height = viewport.height;
    container.appendChild(canvas);
    await page.render({ canvasContext: canvas.getContext('2d'), viewport }).promise;
  }
}

GET /auth/challenge

Gera um nonce de uso único para o endereço informado. O cliente deve assinar a mensagem retornada com MetaMask e enviá-la para /auth/login. O desafio expira em 5 minutos.

POST /auth/login

Verifica a assinatura do desafio e emite um JWT. O token deve ser enviado nas requisições protegidas como Authorization: Bearer <token>.

POST /api/v1/auth/login
Body (JSON)
address * string Endereço Ethereum do usuário
message * string Mensagem exatamente como retornada por /challenge
signature * string Assinatura ECDSA da mensagem (MetaMask / ethers.js)

POST /publisher/encrypt

Cifra um PDF em formato .dlm v3 vinculado a uma chave pública (carteira do comprador). Usado pela editora (Livraria DLM) para gerar os arquivos distribuídos aos compradores.

POST /api/v1/publisher/encrypt
Body (JSON)
pdfBase64 * string Conteúdo do PDF codificado em Base64
publicKey * string Endereço Ethereum do comprador (dono do .dlm)
userName * string Nome completo do comprador (embutido no arquivo)
userCPF * string CPF do comprador (embutido no arquivo)
licenseId string ID da licença (gerado automaticamente se omitido)
title string Título do livro (embutido nos metadados DLMm)
author string Autor do livro (embutido nos metadados DLMm)

Resposta

{
  "dlmBase64":  "base64 do arquivo .dlm v3 gerado",
  "licenseId":  "1234567890",
  "version":    3,
  "size":       12345
}

Uso via APIDLM (js/api.js)

const { APIDLM } = window.DLM;

const result = await APIDLM.encrypt(
  pdfBase64,      // PDF em base64
  publicKey,      // endereço Ethereum do comprador
  userName,       // nome completo
  userCPF,        // CPF
  licenseId,      // opcional
  title,          // título do livro
  author          // autor do livro
);
// result.dlmBase64 → arquivo .dlm para download

POST /decrypt

Decriptografa um arquivo .dlm v2/v3. Requer assinatura MetaMask como prova de posse. Retorna o PDF em Base64 — nunca em disco. Usado pelo DLM-PDF Encriptador.

POST /api/v1/decrypt
Body (JSON)
dlmBase64 * string Conteúdo do arquivo .dlm em Base64
publicKey * string Endereço Ethereum do solicitante
signature * string Assinatura MetaMask da mensagem abaixo
message * string Mensagem assinada: DLM:decrypt:{licenseId}:{timestamp}

Resposta

{
  "pdfBase64": "base64 do PDF decriptografado",
  "licenseId": "1234567890"
}

Erros

403 { "error": "Assinatura inválida ou endereço não confere com o dono do arquivo." }
400 { "error": "Arquivo .dlm inválido ou corrompido." }

Uso via APIDLM (js/api.js)

const { APIDLM } = window.DLM;

const result = await APIDLM.decrypt(
  dlmBase64,   // arquivo .dlm em base64
  publicKey,   // endereço Ethereum
  signature,   // assinatura MetaMask
  message      // mensagem assinada
);
// result.pdfBase64 → PDF para renderizar com PDF.js

POST /transfer

Transfere a posse de um .dlm para outro endereço Ethereum. Re-encripta o arquivo com a chave do novo dono e invalida o .dlm do dono anterior. Requer assinatura MetaMask do remetente.

POST /api/v1/transfer
Body (JSON)
fromPublicKey * string Endereço Ethereum do remetente (dono atual)
toPublicKey * string Endereço Ethereum do destinatário
licenseId * string ID da licença a transferir
signature * string Assinatura MetaMask do remetente
message * string Mensagem assinada

Resposta

{
  "success":      true,
  "licenseId":    "1234567890",
  "newOwner":     "0x...",
  "dlmBase64":    "novo arquivo .dlm para o destinatário"
}

Uso via APIDLM (js/api.js)

// Verificar destinatário antes de transferir
const preview = await APIDLM.previewTransfer(toPublicKey, licenseId);
// preview.toName → nome do destinatário (confirmar com o usuário)

// Executar transferência
const result = await APIDLM.transfer(
  fromPublicKey, toPublicKey, licenseId, signature, message
);
// result.dlmBase64 → novo .dlm para entregar ao destinatário
