Como o DLM-PDF funciona

O DLM-PDF é um sistema de gestão de posse digital de e-books usando blockchain. Cada cópia de livro é um ativo único na blockchain. O leitor precisa provar que é o dono da licença antes de receber a chave para decriptografar o PDF.

Fluxo de autenticação e leitura

1
Conectar carteira (MetaMask)

O browser pede permissão ao MetaMask para acessar o endereço da carteira do usuário.

2
Solicitar desafio

GET /auth/challenge?address=0x... — servidor gera um nonce único com validade de 5 minutos.

3
Assinar com a carteira

MetaMask mostra a mensagem ao usuário. A assinatura prova a posse da chave privada — sem senha, sem cadastro.

4
Login — receber JWT

POST /auth/login — servidor verifica a assinatura e emite um token JWT (validade 1h).

5
Abrir a licença

POST /licenses/:id/open — servidor consulta a blockchain, confirma que o endereço é o dono e retorna a chave de sessão para decriptografar o .dlm.

6
Decriptografar em memória

O cliente usa a chave de sessão + Web Crypto API para decriptografar o .dlm localmente. O PDF nunca vai ao disco.

Formato do arquivo .dlm

O .dlm é um PDF criptografado com um cabeçalho proprietário:

┌──────────────────────────────────────────────────┐
│ MAGIC     4 bytes  "DLM\x01"                     │
│ licenseId 8 bytes  uint64 big-endian              │
│ IV       16 bytes  vetor de inicialização AES     │
│ HMAC     32 bytes  integridade SHA-256            │
│ ciphertext N bytes AES-256-CBC (conteúdo do PDF)  │
└──────────────────────────────────────────────────┘

A chave AES nunca está no arquivo. Ela é derivada por HKDF-SHA256 da chave mestra do servidor + licenseId. Sem a licença válida na blockchain, o arquivo não abre.

Integrar no seu site

Cole o código abaixo no seu projeto para autenticar usuários e abrir arquivos .dlm.

⚙️ Substitua API_BASE pela URL do seu servidor em produção. Em desenvolvimento, use http://localhost:3000/api/v1.

Passo 1 — Incluir dependências

Passo 2 — Autenticar com MetaMask

const API_BASE = 'http://localhost:3000/api/v1';

async function loginComMetaMask() {
  // 1. Solicita acesso à carteira
  const provider = new ethers.BrowserProvider(window.ethereum);
  await provider.send('eth_requestAccounts', []);
  const signer  = await provider.getSigner();
  const address = (await signer.getAddress()).toLowerCase();

  // 2. Pega o desafio do servidor
  const res1 = await fetch(`${API_BASE}/auth/challenge?address=${address}`);
  const { message } = await res1.json();

  // 3. Assina (abre popup MetaMask)
  const signature = await signer.signMessage(message);

  // 4. Faz login e recebe o JWT
  const res2 = await fetch(`${API_BASE}/auth/login`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ address, message, signature }),
  });
  const { token } = await res2.json();

  return { token, address }; // guarda o token para próximas chamadas
}

Passo 3 — Abrir um .dlm e renderizar

// token = JWT recebido no login
async function abrirDLM(token, dlmFile, container) {
  const buffer    = await dlmFile.arrayBuffer();
  const view      = new DataView(buffer);
  const licenseId = (BigInt(view.getUint32(4, false) * 0x100000000) +
                     BigInt(view.getUint32(8, false))).toString();

  // Solicita a chave de sessão (valida posse on-chain)
  const res = await fetch(`${API_BASE}/licenses/${licenseId}/open`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
    },
    body: '{}',
  });
  if (!res.ok) throw new Error('Acesso negado ou licença inválida');
  const { sessionKey } = await res.json();

  // Decripta em memória
  const bytes     = new Uint8Array(buffer);
  const iv        = bytes.slice(12, 28);
  const cipher    = bytes.slice(60);
  const keyBytes  = hexToBytes(sessionKey);
  const cryptoKey = await crypto.subtle.importKey(
    'raw', keyBytes, { name: 'AES-CBC' }, false, ['decrypt']
  );
  const pdfBuffer = await crypto.subtle.decrypt(
    { name: 'AES-CBC', iv }, cryptoKey, cipher
  );

  // Renderiza com PDF.js
  const pdf = await pdfjsLib.getDocument({ data: pdfBuffer }).promise;
  container.innerHTML = '';
  for (let n = 1; n <= pdf.numPages; n++) {
    const page     = await pdf.getPage(n);
    const viewport = page.getViewport({ scale: 1.5 });
    const canvas   = document.createElement('canvas');
    canvas.width   = viewport.width;
    canvas.height  = viewport.height;
    container.appendChild(canvas);
    await page.render({ canvasContext: canvas.getContext('2d'), viewport }).promise;
  }
}

function hexToBytes(hex) {
  const b = new Uint8Array(hex.length / 2);
  for (let i = 0; i < hex.length; i += 2)
    b[i/2] = parseInt(hex.substr(i, 2), 16);
  return b;
}

GET /auth/challenge

Gera um nonce de uso único para o endereço informado. O cliente deve assinar a mensagem retornada com MetaMask e enviá-la para /auth/login. O desafio expira em 5 minutos.

GET /api/v1/auth/challenge
Query params
address * string Endereço Ethereum (0x + 40 hex chars)
Testar agora

Resposta

{
  "message":   "Bem-vindo ao DLM-PDF!\n\nNonce: abc123...\nExpira em: ...",
  "nonce":     "abc123...",
  "expiresAt": 1715000000000
}

POST /auth/login

Verifica a assinatura do desafio e emite um JWT. O token deve ser enviado nas requisições protegidas como Authorization: Bearer <token>.

POST /api/v1/auth/login
Body (JSON)
address * string Endereço Ethereum do usuário
message * string Mensagem exatamente como retornada por /challenge
signature * string Assinatura ECDSA da mensagem (MetaMask / ethers.js)

Resposta de sucesso

{
  "token":   "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "address": "0xf39fd6e51aad88f6f4ce6ab8827279cfffb92266"
}

Erros

401 { "error": "Assinatura inválida: endereço não confere." }
401 { "error": "Desafio expirado. Solicite um novo." }

POST /licenses/:id/open

Endpoint principal do DLM-PDF. Valida a posse da licença na blockchain e retorna a chave de sessão para decriptografar o arquivo .dlm localmente. Requer JWT.

POST /api/v1/licenses/:id/open 🔐 Bearer token
Path params
id * number ID da licença (lido do cabeçalho do .dlm, bytes 4–12)
Testar (precisa de JWT)

Resposta de sucesso

{
  "granted":      true,
  "licenseId":    "1",
  "txHash":       "0xabc...",
  "sessionKey":   "64-char hex — use para decriptografar o .dlm",
  "sessionNonce": "32-char hex",
  "expiresAt":    1715003600000
}

Erros

403 { "error": "Acesso negado. Você não é o proprietário nem o tomador desta licença." }
401 { "error": "Token de autenticação ausente." }
503 { "error": "BLOCKCHAIN_OFFLINE" }

Cifra um PDF em formato .dlm vinculado a uma licença específica. Usado pela editora para gerar os arquivos que serão distribuídos aos compradores. Requer JWT.
